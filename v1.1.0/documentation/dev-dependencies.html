Development Dependencies

Materia relies on several open-source projects to function correctly. The following catalog outlines the primary dependencies and their roles within the project.

Vite

Materia uses Vite as its bundler. Vite is responsible for bundling all the required files and dependencies of the project into a single, optimized file that can be served to the browser.

Following Vite commands are configured in package.json which you can run using npm <command>:

...
"scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
},
...
  • The dev (npm run dev) command runs your app in development mode and starts the local dev server at http://localhost:5173 (default).
  • The build (npm run build) command builds your app for production. It generates a /dist folder with optimized assets.
  • The preview (npm run preview) command serves the built project locally so you can preview it as if it were in production.

The vite.config.js file can be found at the root of the project and it contains the configuration for Vite.

To streamline and enhance the development experience, Beyonder integrates a set of Vite plugins that add powerful capabilities and reduce manual setup.

vite-plugin-handlebars

Materia utilizes vite-plugin-handlebars to process HTML files, particularly when serving partial HTML files using Handlebars.

Example usage:

partials/header.html

<header>
    <h1>This is a page header</h1>
</header>

index.html

<!DOCTYPE html>
<html>
  <body>
    {{> header }}

    <div>...</div>
  </body>
</html>

vite-plugin-live-reload

By default, hot reloading does not work with partial file changes. To address this, we use the vite-plugin-live-reload plugin, which ensures that changes to partials are detected and the browser reloads automatically.

Prettier

Materia uses Prettier to format the code. To make any changes to Prettier’s configuration, the .prettierrc file can be located at the root of the project.

PostCSS

PostCSS is a tool for transforming CSS with JavaScript plugins. In Materia, PostCSS is used to automatically add vendor prefixes to CSS rules, ensuring compatibility across different browsers. The configuration file .postcssrc can be located at the root of the project directory.
